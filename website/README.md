# verba.nuvo.page

The landing page — now in 8 languages. The design tokens in
[`styles.css`](./styles.css) are lifted verbatim from the app's
`src/theme.css`, so the site and the product stay the same object in two places.

```
website/
  index.html        root page — auto-redirects based on browser language
  template.html     HTML template with {{group.key}} tokens
  build.cjs         zero-dependency Node.js build script
  styles.css        tokens + layout + lang-switcher
  strings/          one JSON file per language
    en.json         English (reference — complete)
    tr.json         Turkish (complete)
    es.json         Spanish (complete)
    fr.json         French (placeholder)
    de.json         German (placeholder)
    ja.json         Japanese (placeholder)
    it.json         Italian (placeholder)
    pt.json         Portuguese (placeholder)
  en/               generated output (commit this)
  tr/               generated output (commit this)
  es/ etc.          generated output (commit this)
  assets/           logo, screenshots, OG card
  _headers          cache + security headers (Cloudflare Pages)
  _redirects        /download → GitHub releases
  robots.txt
  sitemap.xml       generated by build.cjs — one <url> per language with hreflang
```

## Adding or updating translations

Edit the JSON file in `strings/`. The keys are stable — every language
file has the same structure. Some values contain HTML (`<b>`, `<em>`,
`<a href="...">`) — keep the tags intact.

After editing, rebuild:

```bash
node build.cjs
```

The build writes one `index.html` per language and regenerates `sitemap.xml`.
**Commit the generated output folders** — Cloudflare Pages deploys them as-is
(no build step on the server).

## Run it locally

```bash
python3 -m http.server 8791 --directory website
# → http://localhost:8791             (redirects)
# → http://localhost:8791/en/         (English)
# → http://localhost:8791/tr/         (Turkish)
```

## SEO notes

- Each language page has `<link rel="alternate" hreflang="...">` for all 8
  languages plus `x-default` (set to `/en/`).
- Canonical URLs are language-specific (`/en/`, `/tr/`, etc.).
- The root `/` page has `<meta name="robots" content="noindex, follow">`
  to avoid duplicate-content penalties — only the language-specific pages
  are indexed.
- `sitemap.xml` includes all languages with `<xhtml:link>` alternates.

## Deploy — Cloudflare Pages

Connect the GitHub repo once in the Cloudflare dashboard
(**Workers & Pages → Create → Pages → Connect to Git**) and give it:

| Setting | Value |
|---|---|
| Production branch | `master` |
| Framework preset | **None** |
| Build command | *(leave empty)* |
| Build output directory | `website` |
| Root directory | `/` |

There is nothing to compile on the server — build locally with `node build.cjs`
and commit the output. Every push to `master` that touches `website/` redeploys;
every other branch gets a preview URL.

Then, under **Custom domains**, add `verba.nuvo.page`. The `nuvo.page` zone is
already on Cloudflare, so the CNAME is created for you.

### Or from the CLI

```bash
npx wrangler pages deploy website --project-name=verba
```

## Keeping it honest

The download links and the version badge are hard-coded to a release tag. When you cut
a new one, update:

- the `v0.1.0` in the hero eyebrow and the download eyebrow,
- the six release URLs in the `#download` section.

The screenshots in `assets/` are copies of `docs/screenshots/`. Recapture there, copy
across, and the OG card (`assets/og.png`) only needs regenerating if the wording on it
changes.
